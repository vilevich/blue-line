Blue Line Design System

OPSWAT's shared design system — tokens, components, and icons for building consistent product UIs. Based on Figma Blue Line 2.2.2.

Installation

Add these links to your HTML <head> to get started:

<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;500;700&display=swap" rel="stylesheet">
<link rel="stylesheet" href="https://vilevich.github.io/blue-line/tokens.css">
<link rel="stylesheet" href="https://vilevich.github.io/blue-line/components.css">
<link rel="stylesheet" href="https://vilevich.github.io/blue-line/icons.css">

File Structure

FilePurpose
tokens.cssCSS custom properties, @font-face, dark mode variable overrides, body base styles
components.cssAll reusable component styles with colocated dark mode overrides
icons.cssSVG icons as CSS mask-images — inherit color from parent via currentColor
tokens.jsonStructured design tokens for Style Dictionary, Figma Tokens, etc.
Load order matters: tokens.css first, then components.css, then icons.css. Product-specific styles go in an inline <style> block after the DS links.

Dark Mode

Set data-theme="dark" on the <html> element. All tokens and components auto-adapt.

<!-- Toggle dark mode -->
<html data-theme="dark">

Tables

Two table types: Small (auto layout, ≤5 columns, even widths) and Big (.table-fixed, many columns, pinned checkbox/name/action, horizontal scroll). All cells use 12px inner padding, 20px outer. Content never wraps — truncates with ellipsis.

Use small tables (.data-table) for <= 5 columns with auto layout. Use big tables (.data-table.table-fixed) for 6+ columns with pinned columns and horizontal scroll.
